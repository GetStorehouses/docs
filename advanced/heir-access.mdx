## Overview

Heir Access is an emergency access feature that allows you to designate trusted contacts who can request access to your portfolio in case of incapacitation or death. This ensures your valuable asset records aren't lost and your beneficiaries can access important information when needed.

## What is Heir Access?

Heir Access provides a secure, time-delayed mechanism for designated individuals to request access to your portfolio. This feature balances security with emergency access needs, ensuring your asset records can be recovered by trusted parties if you become unable to access your account.

## How Heir Access Works

### The Emergency Access Process

<Steps>
  <Step title="You Designate Heirs">
    Set up one or more trusted contacts with their email addresses
  </Step>

  <Step title="Heir Requests Access">
    If needed, your designated heir initiates an access request through Storehouses
  </Step>

  <Step title="Waiting Period Begins">
    A security waiting period starts (default: 30 days)
  </Step>

  <Step title="You're Notified">
    You receive multiple email notifications during the waiting period
  </Step>

  <Step title="You Can Deny Request">
    If you're able, you can immediately deny the request
  </Step>

  <Step title="Access Granted or Denied">
    After the waiting period, if not denied, heir receives read-only access
  </Step>
</Steps>

### Security Features

<AccordionGroup>
  <Accordion title="Time-Delayed Access" icon="clock">
    - Default 30-day waiting period prevents unauthorized access
    - You receive notifications throughout the waiting period
    - Sufficient time to deny requests if you're still active
  </Accordion>

  <Accordion title="Email Notifications" icon="envelope">
    - Notification when access request is initiated
    - Reminders at 7, 14, and 21 days into waiting period
    - Final notification 24 hours before access is granted
    - Notifications include link to immediately deny request
  </Accordion>

  <Accordion title="Read-Only Access" icon="eye">
    - Heirs receive view-only access to your portfolio
    - Cannot modify, delete, or add items
    - Cannot change account settings
    - Can view all items, values, and export data
  </Accordion>
</AccordionGroup>

## Setting Up Heir Access

### Accessing Heir Settings

Navigate to [storehouses.app/heir](https://storehouses.app/heir) or:

1. Go to Account Settings
2. Click "Heir Access" in the sidebar
3. View the Heir Access configuration page

### Adding a Designated Heir

<Steps>
  <Step title="Click 'Add Heir'">
    Click the "Add Designated Heir" button
  </Step>

  <Step title="Enter Contact Information">
    Provide the heir's details:
    - Full name
    - Email address
    - Relationship (optional: spouse, child, sibling, etc.)
    - Notes (optional: why you're designating them)
  </Step>

  <Step title="Set Waiting Period">
    Choose the emergency access waiting period:
    - 14 days (minimum)
    - 30 days (recommended)
    - 60 days
    - 90 days (maximum)
  </Step>

  <Step title="Save Configuration">
    Click "Add Heir" to save
  </Step>

  <Step title="Confirm via Email">
    You'll receive a confirmation email about the new heir designation
  </Step>
</Steps>

<Tip>
  The heir does not receive any notification when you designate them. They only receive communication if they initiate an access request.
</Tip>

### Multiple Heirs

You can designate multiple heirs:

- Each heir is independent
- Each has their own waiting period
- Each can initiate separate access requests
- No limit on number of designated heirs (Pro feature)

**Common Configurations**:

```
Primary: Spouse (30-day waiting period)
Secondary: Adult Child (30-day waiting period)
Tertiary: Executor/Attorney (60-day waiting period)
```

## The Access Request Process

### How Heirs Request Access

When a designated heir needs access:

1. They visit [storehouses.app/heir](https://storehouses.app/heir)
2. Click "Request Emergency Access"
3. Enter your email address (the account holder)
4. Enter their email address (must match designation)
5. Submit the request

### What Happens After Request

**Day 0: Request Initiated**
- You receive immediate email notification
- Heir receives confirmation their request is processing
- Waiting period begins

**Days 1-30: Waiting Period**
- You receive reminder emails at intervals
- Each email includes "Deny Access" button
- Heir receives periodic status updates

**Day 30: Access Decision**
- If you haven't denied: Heir receives read-only access
- If you denied: Request is canceled, heir is notified

### Denying an Access Request

If you receive an heir access request notification and you're able to respond:

1. Click the "Deny Access" link in any notification email
2. Confirm the denial
3. The request is immediately canceled
4. The heir is notified their request was denied
5. A new waiting period is required for any future requests

<Tip>
  You can also deny requests from your account settings at any time during the waiting period.
</Tip>

    - Current portfolio value
    - Purchase prices
    - Market values
    - Insurance values
    - Historical precious metals prices

    - Collections and their contents
    - Storage locations
    - Item organization structure
  </Accordion>

    - Export portfolio to PDF
    - Export data to CSV
    - Generate printed reports
    - Download all data
  </Accordion>

### What Heirs Cannot Access

Heirs with emergency access cannot:

- Add, edit, or delete items
- Modify account settings
- Change email or password
- Access billing information
- Set up or modify alerts
- Delete the account
- Modify collections or locations
- Invite other users
- Revoke their own access

### Access Duration

Once granted, heir access remains until:

- You revoke it manually
- You delete your account
- The heir voluntarily relinquishes access
- 180 days pass (automatic expiration)

<Note>
  Access automatically expires after 180 days to prevent indefinite access. Heirs can request renewal if still needed.
</Note>

## Revoking Access

### Revoking an Active Access Request

During the waiting period:

1. Click "Deny Access" in any notification email, or
2. Go to Heir Access settings
3. Find the pending request
4. Click "Deny Request"
5. Confirm denial

### Revoking Granted Access

If an heir has already received access:

1. Go to Heir Access settings
2. Find the heir with "Access Granted" status
3. Click "Revoke Access"
4. Confirm revocation
5. The heir immediately loses portfolio access

<Note>
  Revoking access doesn't remove the heir designation. They can still request access again in the future.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Heir Can't Request Access" icon="circle-xmark">
    Common issues:
    - Email address doesn't match designation
    - Your account email is incorrect
    - Heir is not actually designated
    - Previous request is still pending

    Solution: Verify heir's email in your settings and confirm they're using the correct account email.
  </Accordion>

  <Accordion title="Not Receiving Notifications" icon="envelope-slash">
    If you don't receive access request emails:
    - Check spam/junk folders
    - Verify your account email is correct
    - Add noreply@storehouses.app to contacts
    - Check email settings in your account

    Contact support immediately if you suspect an unauthorized request.
  </Accordion>

  <Accordion title="Can't Deny Request" icon="hand">
    If denial links aren't working:
    - Log into your account directly
    - Navigate to Heir Access settings
    - Manually deny from the dashboard
    - Contact support if issues persist
  </Accordion>

  <Accordion title="Heir Has Access Issues" icon="user-xmark">
    If a legitimate heir cannot access after approval:
    - Verify the waiting period has fully elapsed
    - Check their email for access instructions
    - Ensure they're logging in correctly
    - Contact support for assistance
  </Accordion>
</AccordionGroup>
